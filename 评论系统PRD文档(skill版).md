## 1. 产品背景

### 1.1 业务背景

在项目协作中，团队成员对数据、方案的沟通分散在微信、口头等多种渠道，存在以下痛点：

| 痛点 | 描述 | 影响 |
| :--- | :--- | :--- |
| 沟通分散 | 讨论内容分散在多个渠道，缺乏统一入口 | 查找困难，信息易遗漏 |
| 缺乏上下文 | 口头沟通后无记录，难以追溯决策过程 | 责任不清，易扯皮 |
| 效率低下 | 关键信息需反复确认，传递链长 | 项目进度受阻 |

### 1.2 解决问题

通过构建统一的评论系统，实现：
- 在具体数据/方案页面直接发起讨论，形成上下文关联
- @相关人员，实现精准消息触达
- 所有沟通记录永久留存，支持历史追溯

## 3. 核心流程

### 3.1 评论发布流程

```
用户进入数据/方案页面
        ↓
    点击评论入口
        ↓
    进入评论区组件
        ↓
    输入评论内容
        ↓
是否 @人员？
    ├── 是 → 选择项目成员 → 发送评论
    └── 否 → 直接发送评论
        ↓
    评论展示在评论区
        ↓
被 @用户收到站内通知
```

### 3.2 评论查看流程

```
用户进入数据/方案页面
        ↓
    点击评论区
        ↓
    系统加载评论列表（分页）
        ↓
    按时间倒序展示
        ↓
    用户可查看嵌套回复
        ↓
    可对评论进行回复操作
```

## 5. 业务规则

### 5.1 评论规则

| 规则编号 | 规则内容 |
| :--- | :--- |
| BR-001 | 评论内容不能为空，最大长度 2000 字符 |
| BR-002 | 评论必须关联具体的数据或方案 |
| BR-003 | 评论发布后不允许修改，只能删除重新发布 |
| BR-004 | 删除评论会同时删除该评论下的所有回复 |

### 5.2 @规则

| 规则编号 | 规则内容 |
| :--- | :--- |
| BR-005 | @功能只能选择当前项目的成员 |
| BR-006 | 单条评论最多@10个用户 |
| BR-007 | 被@用户必须收到站内通知 |

### 5.3 权限规则

| 规则编号 | 规则内容 |
| :--- | :--- |
| BR-008 | 仅项目成员可以查看和发布评论 |
| BR-009 | 评论作者可以删除自己的评论 |
| BR-010 | 项目管理员可以删除任何评论 |

### 5.4 通知规则

| 规则编号 | 规则内容 |
| :--- | :--- |
| BR-011 | 被@或被回复的用户收到通知 |
| BR-012 | 通知未读数在个人中心展示 |
| BR-013 | 点击通知标记为已读 |

## 8. 接口设计

### 8.1 发布评论

- **接口路径**：`POST /api/comments`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| target_type | string | 是 | 目标类型 |
| target_id | number | 是 | 目标ID |
| content | string | 是 | 评论内容 |
| parent_id | number | 否 | 父评论ID（回复时传） |
| mentioned_users | array | 否 | @用户ID列表 |

- **响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "target_type": "data",
    "target_id": 100,
    "user_id": 1,
    "content": "这个方案需要再确认一下",
    "parent_id": null,
    "mentioned_users": [2],
    "created_at": "2026-05-07 10:30:00"
  }
}
```

### 8.2 获取评论列表

- **接口路径**：`GET /api/comments`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| target_type | string | 是 | 目标类型 |
| target_id | number | 是 | 目标ID |
| page | number | 否 | 页码，默认1 |
| size | number | 否 | 每页数量，默认20 |

### 8.3 获取通知列表

- **接口路径**：`GET /api/notifications`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | number | 否 | 页码，默认1 |
| size | number | 否 | 每页数量，默认20 |

### 8.4 标记通知已读

- **接口路径**：`PUT /api/notifications/{id}/read`
