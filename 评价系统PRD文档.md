## 1. 业务背景

在项目协作中，团队成员对数据、方案的沟通分散在微信、口头等多种渠道，存在以下痛点：
- 沟通缺乏上下文关联，难以追溯决策过程
- 重要讨论内容无留存，导致责任不清、易扯皮
- 信息传递效率低，关键消息易遗漏

## 2. 功能目标

| 目标编号 | 目标描述 | 优先级 |
| :--- | :--- | :--- |
| GOAL-001 | 支持在具体数据/方案下发布评论，建立上下文关联 | 高 |
| GOAL-002 | 支持@相关人员，实现精准消息通知 | 高 |
| GOAL-003 | 所有沟通记录永久可追溯 | 高 |
| GOAL-004 | 提供评论回复功能，支持对话式讨论 | 中 |

## 7. 数据字典

| 字段名 | 类型 | 长度 | 约束 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| id | BIGINT | - | PRIMARY KEY | 评论ID |
| target_type | VARCHAR | 50 | NOT NULL | 关联目标类型（data/solution） |
| target_id | BIGINT | - | NOT NULL | 关联目标ID |
| user_id | BIGINT | - | NOT NULL | 发布用户ID |
| content | TEXT | - | NOT NULL | 评论内容 |
| parent_id | BIGINT | - | NULL | 父评论ID（回复时使用） |
| mentioned_users | JSON | - | NULL | @用户列表 |
| created_at | DATETIME | - | NOT NULL | 创建时间 |
| updated_at | DATETIME | - | NOT NULL | 更新时间 |

### 7.2 通知表 (notifications)

| 字段名 | 类型 | 长度 | 约束 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| id | BIGINT | - | PRIMARY KEY | 通知ID |
| user_id | BIGINT | - | NOT NULL | 接收用户ID |
| type | VARCHAR | 50 | NOT NULL | 通知类型（mention/reply） |
| source_id | BIGINT | - | NOT NULL | 来源ID（评论ID） |
| content | VARCHAR | 500 | NOT NULL | 通知摘要 |
| is_read | TINYINT | 1 | DEFAULT 0 | 是否已读 |
| created_at | DATETIME | - | NOT NULL | 创建时间 |

## 8. 接口设计

### 8.1 发布评论

- **接口路径**：`POST /api/comments`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| target_type | string | 是 | 目标类型 |
| target_id | number | 是 | 目标ID |
| content | string | 是 | 评论内容 |
| parent_id | number | 否 | 父评论ID（回复时传） |
| mentioned_users | array | 否 | @用户ID列表 |

- **响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "target_type": "data",
    "target_id": 100,
    "user_id": 1,
    "content": "这个方案需要再确认一下",
    "parent_id": null,
    "mentioned_users": [2],
    "created_at": "2026-05-07 10:30:00"
  }
}
```

### 8.2 获取评论列表

- **接口路径**：`GET /api/comments`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| target_type | string | 是 | 目标类型 |
| target_id | number | 是 | 目标ID |
| page | number | 否 | 页码，默认1 |
| size | number | 否 | 每页数量，默认20 |

- **响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [...],
    "total": 100,
    "page": 1,
    "size": 20
  }
}
```

### 8.3 获取通知列表

- **接口路径**：`GET /api/notifications`
- **请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | number | 否 | 页码，默认1 |
| size | number | 否 | 每页数量，默认20 |

- **响应示例**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [...],
    "total": 50,
    "unread_count": 5
  }
}
```

### 8.4 标记通知已读

- **接口路径**：`PUT /api/notifications/{id}/read`
- **响应示例**：
```json
{
  "code": 0,
  "message": "success"
}
```
